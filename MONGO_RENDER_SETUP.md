# Glutenia MongoDB Atlas, Compass, and Render Setup

Use this checklist for the current production setup:

```text
Mobile app -> https://glutenia.onrender.com/api -> Render Node service -> MongoDB Atlas
```

## Important Security Step

The MongoDB URI contains a database password. If it was pasted anywhere public or semi-public, rotate the Atlas database user's password before treating the project as production-ready.

Do not commit `.env` files. This repo ignores them already.

## MongoDB Atlas

The app expects these Render environment variables:

```env
MONGO_URI=mongodb+srv://USERNAME:PASSWORD@CLUSTER.mongodb.net/?retryWrites=true&w=majority&appName=glutenia
MONGO_DB_NAME=glutenia
JWT_EXPIRES_IN=7d
CORS_ORIGIN=
```

`JWT_SECRET` can be generated by Render from `render.yaml`.

The backend also supports a URI that already includes the database name:

```env
MONGO_URI=mongodb+srv://USERNAME:PASSWORD@CLUSTER.mongodb.net/glutenia?retryWrites=true&w=majority&appName=glutenia
```

If the URI does not include `/glutenia`, `MONGO_DB_NAME=glutenia` tells Mongoose which database to use.

## MongoDB Compass

In Compass, paste the same Atlas connection URI. After connecting, use the `glutenia` database.

Expected collections after seeding:

- `users`
- `products`
- `orders`
- `carts`

Seed data includes:

```text
admin@glutenia.tn
admin123
```

## Render

This repo includes `render.yaml` for a free Render web service:

- Service name: `glutenia`
- Root directory: `glutenia-backend`
- Build command: `npm ci`
- Start command: `npm start`
- Health check path: `/`

In the Render dashboard for the `glutenia` service, set:

```env
MONGO_URI=your Atlas URI
MONGO_DB_NAME=glutenia
NODE_ENV=production
JWT_EXPIRES_IN=7d
CORS_ORIGIN=
```

Render supplies the `PORT` value automatically. The backend binds to `0.0.0.0` so Render can route traffic to it.

## First Production Seed

After Render deploys successfully, open the Render Shell for the service and run:

```bash
npm run seed
```

This creates the admin user and sample products in Atlas. It clears existing users and products, so use it only when you intentionally want to reset seed data.
